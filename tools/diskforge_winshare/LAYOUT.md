# diskforge_winshare_v1 — share layout spec

The disk image simulates a small AD-joined corporate file server's
SMB shares as they'd be seen from an authenticated domain user.
Four logical shares laid out on a single NTFS partition (the
benchmark sweeps file paths regardless of which UNC namespace
they'd map to in production):

```
\\corp-fs01\Departments      → /Departments/...
\\corp-fs01\Public           → /Public/...
\\corp-dc01\SYSVOL           → /SYSVOL/...
\\corp-fs01\IT-Admin         → /IT-Admin/...
```

The image is mounted root = `C:\Shares\` in the diskforge build;
the benchmark sweep reads paths as `/Departments/...` etc.

## Precision-stress noise (the gotchas)

A subset of noise filenames deliberately contains credential-shaped
words in benign contexts. These exist to verify the rules DON'T
over-fire — they're noise files (negative ground truth) but their
names test the precision of the cred-keyword rules:

- `/Departments/HR/policies/password_policy.docx` (negative — policy doc, not creds)
- `/Departments/HR/onboarding/credential_request_template.docx` (negative — empty form)
- `/Departments/HR/onboarding/account_setup_checklist.xlsx` (negative — process doc)
- `/Departments/HR/policies/secrets_management_guidelines.pdf` (negative)
- `/Public/Templates/office/key_management_template.xlsx` (negative)
- `/Departments/IT/projects/secret_santa_2024.xlsx` (negative — really)
- `/Departments/Marketing/rebrand_2024/Q3_credentials_review_meeting.docx` (negative — meeting notes)
- `/Departments/Eng/proj-x/src/AuthCredentialsTest.cs` (negative — unit test code)

Target: ~20 precision-stress filenames distributed across noise.

## OS-template noise (free)

Stauffer's windows10 template auto-creates ~35 stub files (System32
binaries, NTUSER.DAT, registry hives, etc). These are zero-byte
placeholders. Most should score Green; any that score Yellow+ is
a false positive on OS-shape paths.

## Realism assumptions worth flagging

1. **No live AD domain.** Paths use `corp-fs01`/`corp-dc01` as
   placeholder hosts in the layout doc; the disk image has no SMB
   server. The benchmark sweeps file paths in isolation.
2. **No content scanning.** Stage 1 + path rules only. The 80
   positives have format-shaped content (regex-valid) but the
   benchmark scoring uses path/filename targeting, not file body.
3. **Bias toward AD shapes.** SYSVOL / IT-Admin coverage is heavy
   relative to a typical SMB share. That's intentional — high-value
   creds concentrate in IT/admin paths.
4. **Western-language bias except German.** Category #14 is the
   only non-English coverage. v2 could add Spanish / French / Czech.
